# Reference modelli Django

Reference strutturale (campi, FK) per app. La logica sta nei concetti di
[dominio](../domain/).

* [App properties](properties.md) - anagrafica immobile, proprietari, quote, assegnazioni.
* [App billing](billing.md) - addebiti (Receivable), incassi, utenze, spese.
* [App accounting](accounting.md) - contabilità tra proprietari, ledger, settlement.
* [App accounts](accounts.md) - autenticazione, permessi, inviti, impersonation.
* [App notifications](notifications.md) - template, reminder, Web Push, notifiche.
